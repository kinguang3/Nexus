
# 提示词工程 (Prompt Engineering)

## 一、什么是提示词工程

提示词工程是设计和优化与AI语言模型交互的输入提示的过程，旨在引导模型生成高质量、相关且符合预期的输出。它是充分发挥AI模型能力的关键技能。

**核心目标：**
- 明确传达任务要求
- 控制输出格式和风格
- 提高模型响应的准确性和一致性
- 激发模型的特定能力（推理、创作、分析等）

## 二、核心原则

### 1. 清晰明确
- 使用简洁、直接的语言
- 避免模糊或歧义的表述
- 明确说明期望的输出格式

### 2. 具体详细
- 提供足够的上下文信息
- 指定输出长度和结构
- 说明目标受众和使用场景

### 3. 结构化
- 使用分隔符区分不同部分
- 采用编号或列表组织信息
- 明确标注输入和输出示例

### 4. 迭代优化
- 测试多个变体
- 分析模型响应
- 逐步改进提示词

## 三、常用技巧和策略

### Zero-shot 和 Few-shot

**Zero-shot（零样本）：**
- 不提供示例，直接描述任务
- 依赖模型的预训练知识
- 适用于简单、常见的任务

**Few-shot（少样本）：**
- 提供少量高质量示例
- 帮助模型理解任务模式
- 显著提升复杂任务的表现

**示例对比：**
```
Zero-shot:
"解释什么是量子计算"

Few-shot:
"解释概念：
- 人工智能：模拟人类智能的计算机系统
- 机器学习：从数据中学习模式的算法
- 量子计算："
```

### 链式思考 (Chain of Thought)

引导模型逐步推理，展示中间步骤。

**示例：**
```
"一个房间里有10个人，每个人都戴了一顶帽子。帽子有红色或蓝色两种。每个人都能看到别人的帽子，但看不到自己的。现在所有人同时说出自己帽子的颜色。请问：至少有多少人能猜对？

请一步步推理：
1. 首先考虑最简单的情况..."
```

### 角色设定 (Role Prompting)

为模型指定特定角色，影响输出风格和内容。

**示例：**
```
"你是一位资深的软件架构师，擅长用通俗易懂的方式解释复杂技术概念。请解释什么是微服务架构。"
```

### 格式控制

明确指定输出格式，便于后续处理。

**示例：**
```
"请将以下文本总结成要点，使用JSON格式输出：
{
  \"要点\": [\"...\", \"...\"]
}

文本：人工智能正在改变我们生活的方方面面，从医疗诊断到自动驾驶，再到智能家居。"
```

### 约束条件

设定输出的限制条件。

**示例：**
```
"请用不超过100字解释区块链技术，语言要简洁易懂，避免使用专业术语。"
```

## 四、提示词类型

### 1. 指令型
直接下达任务指令
```
"请翻译以下英文句子：Hello, world!"
```

### 2. 问答型
提出具体问题
```
"什么是大语言模型？它是如何工作的？"
```

### 3. 创作型
激发创意生成
```
"写一首关于春天的诗，使用比喻和拟人手法。"
```

### 4. 分析型
要求深入分析
```
"分析以下文章的主要论点，并评价其论证的逻辑性：[文章内容]"
```

### 5. 对比型
比较不同概念
```
"比较监督学习和无监督学习的优缺点。"
```

## 五、评估指标

### 定性评估
- 相关性：输出是否与输入相关
- 准确性：信息是否正确
- 完整性：是否涵盖所有要点
- 清晰度：表达是否清晰易懂
- 风格匹配：是否符合要求的语气和风格

### 定量评估
- 响应时间
- 输出长度
- 特定关键词覆盖率
- 格式符合度

## 六、高级技巧

### 自我一致性 (Self-Consistency)
```
"请给出多种解决方案，并评估每种方案的优缺点："
```

### 思维树 (Tree of Thoughts)
```
"问题：如何提高客户满意度？

请考虑以下分支：
1. 产品质量改进
2. 客户服务优化
3. 价格策略调整

对于每个分支，给出具体建议..."
```

### 反思优化
```
"请先给出初步回答，然后反思回答是否完整准确，如有不足请补充修正。"
```

## 七、常见陷阱

1. **过度复杂**：提示词过长可能导致模型忽略关键信息
2. **信息遗漏**：缺少必要的上下文或约束
3. **歧义表达**：模糊的指令导致不确定的输出
4. **格式混乱**：缺乏结构化导致模型难以理解
5. **假设偏差**：假设模型拥有特定知识或能力

## 八、最佳实践

1. 从简单提示开始，逐步增加复杂度
2. 使用分隔符（```、---、###）组织内容
3. 提供清晰的输入输出示例
4. 指定预期的输出格式
5. 测试多种提示词变体
6. 记录和复用有效的提示词模板

## 九、提示词模板示例

### 总结模板
```
"请总结以下文本的核心内容，不超过5句话：

文本：[输入文本]

总结："
```

### 分类模板
```
"请将以下内容分类到指定类别中：
类别：[类别列表]
内容：[输入内容]
分类结果："
```

### 创意写作模板
```
"请以[主题]为题写一篇[文体]，要求：
- 字数：[X]字
- 风格：[风格描述]
- 要点：[要点列表]

正文："
```
