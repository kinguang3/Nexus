
# 错误日志

## 2026-07-17

### 修复的问题

#### 1. `knowledge_base.py` - 变量名拼写错误
- **文件**: `Data/knowledge_base.py`
- **位置**: 第71行
- **问题**: 导入的是 `length_hint`，但使用时写成了 `len_hint`，导致 `NameError: name 'len_hint' is not defined`
- **修复**: 将 `len_hint` 改为 `length_hint`

#### 2. `knowledge_base.py` - 模块导入错误
- **文件**: `Data/knowledge_base.py`
- **位置**: 第10行
- **问题**: 导入模块名写成了 `langchain_text_splitter`，实际应为 `langchain_text_splitters`（带s），导致 `ModuleNotFoundError`
- **修复**: 将 `langchain_text_splitter` 改为 `langchain_text_splitters`

#### 3. `knowledge_base.py` - 缺少API密钥配置
- **文件**: `Data/knowledge_base.py`
- **位置**: 第62行
- **问题**: `DashScopeEmbeddings` 初始化时缺少 `dashscope_api_key` 参数，导致 `ValidationError`
- **修复**: 添加 `dashscope_api_key` 参数

#### 4. `knowledge_base.py` - 参数名错误
- **文件**: `Data/knowledge_base.py`
- **位置**: 第70行
- **问题**: `RecursiveCharacterTextSplitter` 的参数名写成了 `separator`，新版本应为 `separators`（复数），导致 `TypeError: TextSplitter.__init__() got an unexpected keyword argument 'separator'`
- **修复**: 将 `separator` 改为 `separators`

### 验证结果
代码修复后成功运行，退出码为0（无错误）。

## 2026-07-18

### 安全类问题

#### 5. `knowledge_base.py` - API密钥硬编码泄露
- **文件**: `Data/knowledge_base.py`
- **位置**: 第62行
- **问题**: DashScope API密钥直接写在代码中并推送到公开GitHub仓库，存在安全泄露风险
- **修复**: 将密钥移至 `.env` 文件，代码中通过 `os.environ.get("DASHSCOPE_API_KEY")` 读取，更新 `.gitignore` 确保 `.env` 不上传

- **问题**: 项目无 `requirements.txt`，协作时难以复现环境

- **位置**: 第8行
